Our production repo must include:

  1. an active license, (Allows others outside of the group to know how they may use our project.)
  2. a README.md file, (An Introduction to our project that describes what it is and what it is for.)
  3. a CONTRIBUTING.md file, (Outlines what the rights and responsibilities are for people making any changes to the project.)
  4. a project page (this site)
  5. a bug reports project board, (a place to list new bugs and the progress on resolving them).
  6. a Super-linter workflow, ( An ingenious plug-in type feature that runs when a pull request is made and lets the user know if there are issues found in the code.)
  7. and all project assets. (Webpages and images)
