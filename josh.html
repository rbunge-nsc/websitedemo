Process Decisions

When working in a group, there are a lot of decision points. In this project, because we could build the website to be what ever we wanted, we were one of the clients as well as the deliverables team. Without the focus of an outside client for the content, and to meet the expectations of the what was required for the class, this was an early sketch of what was needed to set the scope of the project.

- Decide on our roles:
      Software Development,
      Testing,
      Deployment,
      Other:
- Website information? :
      Individual team pages (agreed to by group discussion),
      Corresponding SDLC step through pages?
- Communication tool(s) :
      Github communications(agreed to by group discussion),
      Weekly written reports
- Page design :
      how to set up styles? A template?
- Week 5: Document Workflow :
      Initial Project Design Process (stages 1-3),
      Server Design,
      Software Development (stage 4 and 7),
      Testing (stage 5),
      Deployment (stage 6)
- Week 6: Code Testing :
      Install Super-Linter,
      Upload an error,
      See if it catches
- Week 7: Stack decisions :
      Monolith? 3 tier?... ,
      Which web server did you install?,
      What operating system is it running on?
- Week 8: Changes and Fixes :
      Backups?
- Week 9 – Deployment
      Maintenance? What does it mean to maintain the code? Or is this server maintenance?
      Where did you do the installation? (local machine, cloud provider, virtual machine, other),
      Edit files right on your server directly.
      Use FTP to transfer files manually.
      Use SFTP to transfer files manually.
      Use a version control tool like Git, Subversion (“SVN”) or Mercurial.
      Install version control on the server, pull files manually.
      Set up automated pull from the server.
      Use 3rd party deployment tools.
